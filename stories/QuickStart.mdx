# Quick Start

## 1. Extend Layer in Nuxt Config

After installation, add the design system as a layer in `nuxt.config.ts`:

```typescript
export default defineNuxtConfig({
  extends: ["@tdm/design-system-web"],
});
```

### If Installing from GitHub or Local Path

Use the appropriate path:

```typescript
// Via GitHub
export default defineNuxtConfig({
  extends: ["github:makassar-superapp/design-system-web#main"],
});

// Via Local Path
export default defineNuxtConfig({
  extends: ["../design-system-web"],
});
```

## 3. Use Utilities

The design system provides useful utility functions:

### cn() - Classname Merger

The `cn()` utility for merging classnames with conditional logic:

```vue
<script setup lang="ts">
import { cn } from "@tdm/design-system-web/lib/utils";

const isActive = ref(true);
const classes = cn(
  "base-class",
  isActive.value && "active-class",
  "another-class"
);
</script>
```

### Example Usage in Component

```vue
<template>
  <div :class="containerClasses">
    <Button :class="buttonClasses" @click="toggle"> Toggle </Button>
  </div>
</template>

<script setup lang="ts">
import { cn } from "@tdm/design-system-web/lib/utils";

const isExpanded = ref(false);

const containerClasses = computed(() =>
  cn(
    "p-4 rounded-lg transition-all",
    isExpanded.value ? "bg-primary text-white" : "bg-gray-100"
  )
);

const buttonClasses = computed(() =>
  cn("w-full", isExpanded.value && "bg-white text-primary")
);

const toggle = () => {
  isExpanded.value = !isExpanded.value;
};
</script>
```

## Development Tips

### 1. Restart Dev Server After Setup

After adding the layer to `nuxt.config.ts`, restart the dev server:

```bash
# Kill and restart
pnpm run dev
```

### 2. Restart IDE for IntelliSense

To get autocomplete and type hints, restart your IDE after initial setup.

### 3. Check Component Auto-import

If components are not auto-imported:

1. Ensure the layer is properly extended in `nuxt.config.ts`
2. Clear cache: `rm -rf .nuxt node_modules/.cache`
3. Restart dev server
